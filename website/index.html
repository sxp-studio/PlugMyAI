FAQ

Is this free?

Yes. PlugMyAI is free and open source (MIT). You use your own existing AI subscriptions — there's nothing to pay for.

Is my data private?

PlugMyAI runs entirely on your machine. No data passes through our servers — requests go directly from localhost to your AI provider. All configuration and data is stored locally in ~/.plug-my-ai/.

Is this secure?

By default, PlugMyAI only exposes chat completions — no tools, no code execution, no file access. This means connected apps can only send and receive text. You can optionally enable full access for specific apps, but be mindful: that gives them tool use, which depending on the provider could mean access to your filesystem, shell, and more. Only enable full access for apps you trust.

Does this violate provider terms?

PlugMyAI routes through your own subscriptions. It doesn't share accounts, bypass rate limits, or resell access. You're responsible for your provider's ToS.

What providers are supported?

Claude Code and Codex subscriptions, Ollama for local models, with more coming. The provider system is extensible.

What about Linux and Windows?

macOS only for now. Linux and Windows support could happen if there's interest — pull requests are welcome on GitHub!

Does this work on mobile?

Not out of the box. PlugMyAI binds to localhost, so it's only reachable from the Mac it runs on. If you know what you're doing, you could expose it on your local network and connect from a mobile device — but that's not a supported setup yet.

How do apps connect?

Apps pair with PlugMyAI via a one-time approval dialog. Once approved, they get a token to make OpenAI-compatible API requests on localhost.
